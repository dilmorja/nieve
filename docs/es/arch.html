Introducción

La Arquitectura de software es el diseño de más alto nivel de la estructura de un sistema. Consiste en un conjunto de patrones y abstracciones coherentes que proporcionan un marco definido y claro para interactuar con el código fuente del software.
La documentación describe de forma detallada cada una de las capas y de los niveles que conforman la arquitectura de este proyecto. De esto de obtienen bases sólidas y escalables que serán útiles durante toda la etapa de desarrollo.

Modelo

Existen modelos predefinidos, lo cual nos permite agilizar el proceso de planificación del proyecto. Uno de estos modelos es la proramación por capas.
Las bases de esta arquitectura permanecen en el diseño, lo demás es ajustado a las necesidades del proyecto.

La primera versión del modelo de arquitectura cuenta con tres niveles / pisos y cinco capas:

  1. Primer nivel.
    • Interface o presentación.
    • Drivers o controladores.
  2. Segundo nivel.
    • Logistics o logística.
  3. Tercer nivel.
    • Recursos de datos descentralizados (DDR, por sus siglas en inglés).
    • Database o base de datos.

Definiciones

Nivel o piso.
Es un conjunto de capas que conforman la infraestructura lógica del proyecto.
Capa.
Es un segmento lógico que alivia la carga de trabajo de las demás capas, realizando tareas por independiente. Las capas pueden tener intercomunicación por medio de pipelines y risers.
Pipeline o tubería.
Una tubería es cualquier medio por el que las capas de diferentes niveles se puedan comunicar entre si. Esta comunicación puede ser unidireccional o bidireccional
Riser o elevador.
Un elevador es cualquier medio por el que se envía y / o recibe información de las capas que conforman un nivel. Generalmente tienen puntos de intersección denominados sectores logísticos.
